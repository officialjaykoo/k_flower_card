맞고 룰 페이지 (개발판)

지역/집마다 다른 룰을 구분해 관리합니다. 기본 룰을 기준으로 변형 룰을 추가합니다.

기본 규칙 (12월 화투 48장 + 보너스 2장 = 50장)

  • 딜: 1인 10장, 보드 8장 공개, 남은 22장은 덱.
  • 턴: 손패 1장 냄 → 보드 월 매치 시 캡처, 아니면 보드에 남김 → 덱 1장 뒤집어 동일 처리.
  • 따닥/뻑/쪽: 같은 월 3장 상황은 룰 설정에 따라 처리. 기본 구현은 이벤트 보너스만 부여.
  • Go/Stop: 기본 점수 3점 이상 시 선택 가능(룰 세트별 상이). 3고 제한은 룰 세트에 따름.
  • 종료: 덱/손패 소진 시 정산. Stop 선언 시 즉시 정산.

보너스 카드 (월 13으로 묶음)

  • 보너스쌍피: 쌍피 값 + 효과로 상대 피 1장 훔치기.
  • 보너스트리플피: 트리플피 값 + 효과로 상대 피 1장 훔치기.
  • 훔치기 규칙: 상대 피가 없으면 0. 쌍피/트리플피만 있어도 그 카드를 가져온다(피 가치 높은 순).
  • 카드 파일명 예시: 13-보너스쌍피.png, 13-보너스트리플피.png

점수 계산 (개발 기본)

  • 광: 3장 3점, 4장 4점, 5장 15점.
  • 열끗/고도리: 5장부터 1점씩, 고도리 완성 시 추가 3점.
  • 띠: 5장부터 1점씩, 홍단/청단/초단 완성 시 각 3점.
  • 피: 10피부터 1점씩. 쌍피는 2, 트리플피는 3으로 계산.
  • 보너스: 따닥/뻑/쪽은 룰 세트별 보너스 점수 적용.
  • 배수: 맞고, 광박, 피박, 멍텅구리 등은 룰 세트별 배수 적용.

지역/집 룰 선택지 (앱 내 셀렉트와 연결)

A 표준(방송/대회) 근접

  • Go/Stop: 3점 이상, 최대 3고. 맞고 배수 2배.
  • 따닥/뻑/쪽 보너스 각 1점.
  • 박 배수: 광박/피박/멍텅구리 2배.

B 가족/친목 캐주얼

  • Go/Stop: 3점 이상, 2고 제한.
  • 따닥/뻑/쪽 보너스 각 1점.
  • 박 배수: 광박/피박 2배, 멍텅구리 배수 없음.

C 빠른 단판

  • Go/Stop 없음. 덱 소진 시 바로 정산.
  • 보너스/박 배수 없음.

패기보/AI 참고

  • 패기보: 월별 남은 장 수를 실시간 계산해 표시.
  • AI: 매치 우선 + 남은 장수 기반 기대값으로 카드 선택, Go/Stop은 리드/덱 잔량 고려.

개발 노트

  • 이미지 경로: /cards/<월>-<이름>.png 형식 (예: 3-광.png).
  • 룰 변화가 있을 때 이 페이지 섹션을 복사해 새 변형안으로 추가.
  • 빌드 없이 정적 HTML/ESM로 동작하므로 웹서버에서 바로 테스트 가능.
